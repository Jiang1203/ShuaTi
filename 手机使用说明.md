# 手机使用

## 同一 Wi-Fi 临时使用

1. 电脑双击 `启动手机访问.bat`。
2. 保持命令窗口开启。
3. 手机和电脑连接同一个 Wi-Fi。
4. 在手机浏览器打开 `http://电脑IPv4地址:8765`。

Windows 防火墙首次询问时，需要允许 Python 在专用网络中通信。

这种方式适合在家中临时使用，但普通局域网 HTTP 页面不能安装为 PWA。

## 安装到手机主屏幕

将整个文件夹部署到支持 HTTPS 的静态网站服务，例如 GitHub Pages、Cloudflare Pages 或 Netlify。

- Android Chrome：打开网站后点击页面右上角的安装图标，或浏览器菜单中的“安装应用”。
- iPhone Safari：点击“分享”，然后选择“添加到主屏幕”。

题库、错题、收藏和学习进度保存在当前手机浏览器的 LocalStorage 中。清理浏览器网站数据会同时清除这些内容。

## GitHub 题库同步

静态网页本身不能共享电脑和手机的 LocalStorage。要让电脑端生成的新题库同步到手机，需要把 `data/shared-state.json` 一起上传到 GitHub，并在网页的“专项 > GitHub 题库同步”里填写：

- GitHub 用户名
- 仓库名
- 分支，通常是 `main`
- 数据路径，保持 `data/shared-state.json`
- GitHub Token，仅保存在当前电脑浏览器

Token 建议使用 Fine-grained personal access token，只给这个仓库的 Contents 读写权限。电脑端点击“发布当前题库”后，应用会更新仓库里的 `data/shared-state.json`。手机端打开 GitHub Pages 页面时会自动拉取这个文件；GitHub Pages 通常需要 1-2 分钟才会刷新。

同步只发布题库，不发布电脑端的答题记录、错题、收藏和 API Key。手机端的练习进度仍保存在手机本地。

如果手机曾经打开过旧版本，发布新版后请在手机浏览器刷新页面；仍然乱码时，清除该网站的缓存/网站数据后重新打开。

## 注意

DeepSeek API Key 会从手机浏览器直接发送到 DeepSeek API。只有勾选“仅在本机记住 API Key”时，Key 才会保存在当前设备的 LocalStorage 中。
